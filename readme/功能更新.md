# ApolloTab 功能更新记录

## v1.2.0 (2026-06-30)

### 新增
- **节拍器功能**: 完整支持 GTP 歌曲模式和手动模式的节拍器 MIDI 事件生成
  - **新增 `metronome.py`** (`audio/metronome.py`):
    - `MetronomeConfig`: 节拍器配置类（enabled/volume/numerator/denominator）
    - `MetronomeGenerator`: 节拍器事件生成器，支持两种模式：
      - `generate_from_song()`: 从 GTP 歌曲自动提取 BPM 和拍号
      - `generate_simple()`: 手动指定 BPM、拍号和总时长
  - **通道隔离**: 使用 MIDI 通道 15，避免与旋律轨(0-8/10-14)和鼓轨(9)冲突
  - **音色通用**: 采用 GM 标准木鱼音色 (76 Low Woodblock / 77 High Woodblock)
  - **事件规则**: 重拍(第一拍)使用 High Woodblock(77)，普通拍使用 Low Woodblock(76)，每个点击持续约50ms

- **`MidiConverter` 节拍器支持**: `convert()` 方法新增 `metronome_config` 参数，可将节拍器事件混入主音轨事件序列

- **`GTPPlayer` 节拍器控制**: 新增节拍器控制接口与状态管理，包括启用/禁用、音量调整等

## v1.1.0 (2026-06-29)

### 新增
- **GP7/GP8 鼓轨 Articulation 解析**: 完整支持打击乐演奏法映射到 GM MIDI 音高
  - **新增 `PercussionArticulation` 数据类** (`models/track.py`): 映射 GP7/GP8 `<Articulation>` 节点到 MIDI note
  - **`gpif_parser.py`**: 解析 `<Articulations>` 节点，提取 `instrument` / `label` / `articulation_name` 等字段
  - **`midi_converter.py`**: 鼓轨音符转换时通过 `percussion_articulations` 查找对应 MIDI 音高，替代默认鼓组映射

- **BendType / BendStyle / VibratoType 枚举** (`utils/constants.py`):
  - 对齐 alphaTab 数据结构，完善推弦/揉弦处理
  - `BendType`: Normal / Prebend / Release 等推弦类型
  - `BendStyle`: 推弦绘制风格
  - `VibratoType`: Slight / Wide 等揉弦幅度类型

### 修复
- **鼓组音色失效**: `synth_engine.py` 中 CC#0 发送超出范围的 128，导致 FluidSynth 忽略鼓组切换。改为发送合法的 CC#0=1 和 CC#32=0
- **Bank Select 非法值**: `midi_converter.py` 中鼓组 Bank 128 拆分为合法的 14-bit Bank Select (CC#0 + CC#32)
- **通道音色强制覆盖**: 移除 `player.py` 中 `rebuild_audio_events` 强制覆盖通道音色的逻辑，改由 `midi_converter` 统一处理音色事件，避免音色被意外重置

## v0.5.0 (2026-06-20)

### 新增
- **反复记号(Repeat Signs)展开支持**: 完整实现 Guitar Pro 文件中反复记号的展开与播放同步
  - **`MidiConverter.expand_measure_indices()` 静态方法** ([midi_converter.py:132](file:///e:/Projects/ApolloTab/ApolloTab/audio/midi_converter.py#L132)):
    - 基于栈(Stack)数据结构的嵌套反复展开算法
    - 支持 `||:` (反复起始) 和 `:||n` (反复结束, n=重复次数)
    - 支持任意深度的嵌套反复（栈结构天然支持）
    - 返回实际播放顺序的原始小节索引序列

  - **简单反复示例**: `||: A B :||2 C` → 展开为 `[0, 1, 0, 1, 3]`
  - **嵌套反复示例**: `A ||: B C :||2 D :||2 E` → 展开为 `[0, 1, 2, 3, 2, 3, 4, 1, 2, 3, 2, 3, 4, 5]`

### 同步
- **时间线与 MIDI 事件同步** ([player.py:1388](file:///e:/Projects/ApolloTab/ApolloTab/player.py#L1388)):
  - `init_audio()` 时计算并缓存 `_expanded_measure_indices` 展开序列
  - `build_timeline()` 使用相同序列同步重建时间线
  - 时间线展开原理:
    1. 按 `global_meas_idx` 分组原始时间线条目
    2. 计算每个小节的时长(ms)
    3. 按展开序列重新排列+复制条目
    4. 复制条目的 `scroll_y` 保持不变(视觉位置), `time_ms` 按运行时间累增(音频进度)
  - 反复段播放时，光标会"跳回"之前的视觉位置（正确行为），但音频时间持续增长

| 文件 | 变更说明 |

| `audio/midi_converter.py` | 新增 expand_measure_indices() 静态方法；convert()/convert_all_tracks() 调用展开；更新头部注释和详细文档 |
| `player.py` | 新增 _expanded_measure_indices 缓存；init_audio() 计算展开；build_timeline() 时间线同步逻辑；更新头部注释 |

## v0.4.2 (2026-06-19)

### 新增
- **Linux 多音频驱动自动尝试**: `synth_engine.py` 音频驱动初始化重构为优先级列表模式
  - **原理**: 不同 Linux 发行版/桌面环境使用不同音频后端，按优先级依次尝试直到成功
  - **Windows 驱动链**: `dsound` → `waveout` → `default`
  - **macOS 驱动链**: `coreaudio` → `default`
  - **Linux 驱动链**: `pulseaudio` → `pipewire` → `alsa` → `jack` → `default`
    - pulseaudio: Ubuntu 20.04 及更早版本默认
    - pipewire: Ubuntu 22.04+ / Fedora 34+ 默认(兼容 PulseAudio API)
    - alsa: 纯 ALSA 系统(Docker 容器/无桌面环境)
    - jack: 专业音频工作站
    - default: FluidSynth 内置自动检测(最后兜底)
  - 每次尝试输出日志（成功/失败/异常），便于排查问题

### 优化
- **失败诊断信息增强**: 所有音频驱动均不可用时，输出详细原因和安装指引
  - 可能原因：未安装音频服务、Docker 未挂载 /dev/snd、SSH 无音频转发
  - 各发行版安装命令：Ubuntu(apt)、Fedora(dnf)、Docker(--device /dev/snd)

### 修改文件
| 文件 | 变更说明 |
|------|---------|
| `audio/synth_engine.py` | 驱动初始化重构为优先级循环；增强失败诊断信息；更新头部注释 |

## v0.3.9 (2026-06-14)

### 修复
- **空白小节点击定位不准**: 修复 `build_timeline()` 中空小节(无拍/音符)被 `continue` 跳过的问题
  - 原因：原代码对无 beats 的空白小节直接跳过，导致该小节无 timeline 条目，用户点击时定位到相邻有音符的小节，产生较大位置偏差
  - 影响：含休止符/空小节的谱子，点击这些区域时光标跳到错误位置
  - 方案：为每个空白小节生成占位条目(placeholder)，记录正确的 `scroll_y` 和 `time_ms`

### 新增
- **全局唯一小节ID (`global_meas_idx`)**: 时间线索引新增跨系统/跨页的全局唯一小节标识
  - 每处理完一个小节递增一次，确保 ID 在整首曲子中不重复
  - 支持通过 `find_measure_at_time()` 精确按全局ID回溯定位到任意小节的起始位置
- **`find_measure_at_time()` 方法**: 根据时间或滚动位置查找对应小节的详细信息
  - 返回 `{'global_meas_idx', 'meas_idx', 'start_time_ms', 'start_scroll_y'}`
  - 用于 UI 层实现"点击谱面 → 跳转到对应小节播放"的功能

### 优化
- **哨兵点索引继承**: 时间线末尾的哨兵点现在继承最后真实条目的完整小节索引信息（而非硬编码为0）
  - 修复 B点设置在谱子末尾时 `_find_measure_at_time` 返回 meas_idx=0 的错误

### 修改文件
| 文件 | 变更说明 |
|------|---------|
| `player.py` | 新增 find_measure_at_time()；build_timeline() 重构(空白小节占位+全局ID+哨兵修复)；更新头部注释 |

## v0.3.7 (2026-06-13)

### 修复
- **GTPPlayer 缺少 set_theme 方法**: 修复 `'GTPPlayer' object has no attribute 'set_theme'` 错误
  - 原因：v0.2.6 在 TabRenderer 中添加主题系统时，未在门面类 GTPPlayer 中同步暴露接口
  - 影响：用户无法通过 player 对象直接切换渲染主题

### 新增
- **GTPPlayer 主题管理接口**: 新增 3 个委托方法，完整暴露 TabRenderer 的主题功能
  - `set_theme(theme)`: 切换渲染主题（支持字符串/ThemeConfig 实例）
  - `current_theme_name` (property): 获取当前主题名称
  - `get_available_themes()`: 获取所有可用预设主题列表
  
### 设计模式
- **委托模式(Delegate)**: GTPPlayer 将主题设置请求转发给内部 TabRenderer 实例
- **门面模式(Facade)**: 对外提供统一简化的接口，隐藏内部复杂性

### 使用示例
```python
from ApolloTab.player import GTPPlayer

player = GTPPlayer()
player.load("song.gp5")

# 切换到黑白主题（适合打印）
player.set_theme("light")
images = player.render_track(0)  # 使用新主题渲染

# 查看当前主题
print(player.current_theme_name)  # 输出: "light"

# 查看所有可用主题
print(player.get_available_themes())  # 输出: ['light', 'dark']
```
